### Kontrakt gRPC (`proto/vehicle.proto`)

Definicja serwisu oraz struktur danych.

```proto
syntax = "proto3";

package vehicle;

service VehicleService {
  rpc GetVehicles (GetVehiclesRequest) returns (GetVehiclesResponse);
  rpc GetVehicleById (GetVehicleByIdRequest) returns (Vehicle);
  rpc CreateVehicle (CreateVehicleRequest) returns (Vehicle);
  rpc UpdateVehicle (UpdateVehicleRequest) returns (Vehicle);
  rpc DeleteVehicle (DeleteVehicleRequest) returns (DeleteVehicleResponse);
}

message Vehicle {
  int32 id = 1;
  string make = 2;
  string model = 3;
  int32 year = 4;
  float fuel_tank_capacity = 5;
}

message GetVehiclesRequest {
  int32 limit = 1;
  int32 offset = 2;
}

message GetVehiclesResponse {
  repeated Vehicle rows = 1;
  int32 total = 2;
}

message GetVehicleByIdRequest {
  string id = 1;
}

message CreateVehicleRequest {
  string make = 1;
  string model = 2;
  int32 year = 3;
  float fuel_tank_capacity = 4;
}

message UpdateVehicleRequest {
  string id = 1;
  optional string make = 2;
  optional string model = 3;
  optional int32 year = 4;
  optional float fuel_tank_capacity = 5;
}

message DeleteVehicleRequest {
  string id = 1;
}

message DeleteVehicleResponse {
  bool success = 1;
}

```

### Komendy specyficzne dla gRPC

Jeśli zdecydujesz się na generowanie statycznych typów TypeScript (zalecane dla senior developerów), użyj poniższego skryptu.

**Generowanie kodu z `.proto`:**

```bash
# Tworzenie katalogu na wygenerowany kod
mkdir -p ./src/generated

# Generowanie plików JS i d.ts
grpc_tools_node_protoc \
    --js_out=import_style=commonjs,binary:./src/generated \
    --grpc_out=grpc_js:./src/generated \
    --plugin=protoc-gen-grpc=`which grpc_tools_node_protoc_plugin` \
    -I ./proto ./proto/*.proto

# Generowanie definicji TypeScript
protoc \
    --plugin=protoc-gen-ts=./node_modules/.bin/protoc-gen-ts \
    --ts_out=grpc_js:./src/generated \
    -I ./proto ./proto/*.proto

```

**Uruchomienie serwera:**

```bash
node --loader ts-node/esm src/grpcServer.ts

```

Pliki specyficzne dla gRPC mają być w folderze `src/grpc`.
